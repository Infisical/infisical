## Templating

Use Go templates to transform secrets before storing them in the managed Kubernetes resource. When a template is set, only the keys defined in `template.data` are included in the output.
When no template is set, all fetched secrets are included as-is.

| Field           | Type              | Required | Description                                                   |
| --------------- | ----------------- | -------- | ------------------------------------------------------------- |
| `engineVersion` | string            | Yes      | The template engine version. Must be `v1`.                    |
| `data`          | map[string]string | Yes      | Map of output key names to Go template expressions.           |

Each secret is available in the template context as `.SECRET_KEY`, which is an object with two accessors:
- `.Value` — the secret's value.
- `.SecretPath` — the path of the secret in Infisical.

```yaml
targets:
  - name: my-app-config
    namespace: default
    kind: Secret
    creationPolicy: Owner
    template:
      engineVersion: v1
      data:
        DATABASE_URL: "postgres://{{ .DB_USER.Value }}:{{ .DB_PASS.Value }}@{{ .DB_HOST.Value }}:5432/{{ .DB_NAME.Value }}"
        REDIS_URL: "redis://{{ .REDIS_HOST.Value }}:6379"
```
