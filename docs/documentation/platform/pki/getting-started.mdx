Infisical Certificate Manager helps you issue and manage X.509 certificates across your organization. This guide will help you get started based on what you need to accomplish.

## How Certificate Manager Works

Product admins set up the infrastructure, teams issue certificates:

- **[Certificate Authority](/documentation/platform/pki/ca/overview)** — The entity that signs certificates. Use an internal CA or connect an external one (DigiCert, Let's Encrypt, AWS PCA).
- **[Certificate Policy](/documentation/platform/pki/settings/policies)** — Rules for what certificates can contain: allowed domains, validity periods, key algorithms.
- **[Certificate Profile](/documentation/platform/pki/settings/profiles)** — Combines a CA + policy into a reusable template.
- **[Application](/documentation/platform/pki/applications/overview)** — Where teams work. Groups certificates for a service and configures enrollment (API, ACME, EST, SCEP).

## Who manages what?

| Role | Responsibilities |
|------|------------------|
| **Product Admin** | Set up CAs, policies, profiles. Create Applications and assign team members. |
| **Application Admin** | Configure enrollment methods, alerting, syncs within their Application. |
| **Application Operator** | Issue, renew, and revoke certificates. |

<Info>
  **Product Admin?** Start with [Certificate Authorities](/documentation/platform/pki/ca/overview), then create [Applications](/documentation/platform/pki/applications/overview) for your teams.
  
  **Team member?** Ask your product admin to create an Application and assign you.
</Info>
