Certificate alerting keeps your team informed about critical lifecycle events. Configure alerts on your Application to receive notifications via Slack, PagerDuty, email, or webhooks.

## Alert Types

<CardGroup cols={2}>
  <Card title="Expiration" icon="clock">
    **Scheduled** — Notifies when certificates approach expiration (e.g., 30 days before expiry).
  </Card>
  <Card title="Issuance" icon="certificate">
    **Real-time** — Notifies when a new certificate is issued.
  </Card>
  <Card title="Renewal" icon="arrows-spin">
    **Real-time** — Notifies when a certificate is renewed.
  </Card>
  <Card title="Revocation" icon="ban">
    **Real-time** — Notifies when a certificate is revoked.
  </Card>
</CardGroup>

## Create an Alert

In your Application, navigate to **Alerting** and click **Create Certificate Alert**.

![Create alert](/images/platform/pki/alerting/alert-create.png)

![Alert configuration](/images/platform/pki/alerting/alert-create-modal.png)

### Configuration

| Field | Description |
|-------|-------------|
| **Alert Type** | Expiration, Issuance, Renewal, or Revocation |
| **Alert Name** | A slug-friendly name like `tls-expiry-alert` |
| **Description** | Optional context about this alert |

### Expiration Alert Settings

For expiration alerts, configure when notifications trigger:

| Field | Description |
|-------|-------------|
| **Alert Before** | Time before expiration (e.g., `30d`, `2w`, `3m`) |
| **Daily Alerts** | If enabled, sends daily reminders until certificate expires or is renewed |

<Warning>
  For certificates that cannot be renewed through Infisical, daily alerts continue until the original expiration date.
</Warning>

### Filters

Scope alerts to specific certificates using filters:

| Filter Field | Example | Effect |
|--------------|---------|--------|
| Common Name | Contains `api.` | Only alert on API certificates |
| Profile | Equals `production` | Only alert on production certificates |
| Issuer | Contains `DigiCert` | Only alert on DigiCert-issued certificates |

<Note>
  Event-driven alerts (Issuance, Renewal, Revocation) fire in real-time. Expiration alerts are checked on a daily schedule.
</Note>

## Common Use Cases

<AccordionGroup>
  <Accordion title="Alert on expiring production certificates">
    - **Alert Type**: Expiration
    - **Alert Before**: `30d`
    - **Filter**: Profile equals `production`
    - **Channel**: Slack + PagerDuty
    
    Gives your team a month's notice before any production certificate expires, with escalation to on-call.
  </Accordion>
  <Accordion title="Audit all certificate issuance">
    - **Alert Type**: Issuance
    - **Filter**: None (all certificates)
    - **Channel**: Webhook → your SIEM
    
    Sends every certificate issuance event to your security tooling for audit logging.
  </Accordion>
  <Accordion title="Notify on unexpected revocations">
    - **Alert Type**: Revocation
    - **Filter**: None (all certificates)
    - **Channel**: Slack
    
    Alerts the team immediately when any certificate is revoked so they can investigate.
  </Accordion>
</AccordionGroup>
