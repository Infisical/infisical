This page covers the end-to-end setup for connecting Fortanix Data Security Manager (DSM) to Infisical through an HSM Connector. Read [HSM Connectors](/documentation/platform/pki/settings/hsm-connectors) first for the concept.

## Prerequisites

- A Fortanix DSM account with at least one group and an application configured for PKCS#11 access.
- A host inside your network that can reach your DSM endpoint over HTTPS, where you can install the Fortanix PKCS#11 client and run an [Infisical Gateway](/documentation/platform/gateways/overview).
- Project-level permission to create HSM Connectors in Infisical (`HSM Connectors > Create`).

## Step 1: Install the Fortanix PKCS#11 driver on the Gateway host

Download the Fortanix PKCS#11 client from the Fortanix DSM dashboard (Settings -> Client configurations) for the OS that will run the Gateway. Install it and confirm the driver file path is readable by the Gateway user:

```bash
ls -l /opt/fortanix/pkcs11/fortanix_pkcs11.so
```

The default install path on Linux is `/opt/fortanix/pkcs11/fortanix_pkcs11.so`. If you put it somewhere else, use that path everywhere in the steps below.

Create or copy the Fortanix client configuration file at `/etc/fortanix/pkcs11/pkcs11.conf`. The minimum keys are the DSM endpoint and the API key for the application that will be used as the PKCS#11 user. Fortanix provides the file content directly from the dashboard.

Verify the driver loads:

```bash
pkcs11-tool --module /opt/fortanix/pkcs11/fortanix_pkcs11.so --list-slots
```

You should see one slot per Fortanix DSM group your application is a member of, each with a slot label and a serial number.

## Step 2: Start the Gateway with the driver attached

Append `--pkcs11-module=/opt/fortanix/pkcs11/fortanix_pkcs11.so` to the Gateway start (or systemd install) command:

<Tabs>
  <Tab title="Linux (systemd)">
    ```bash
    sudo infisical gateway systemd install <gateway-name> \
      --enroll-method=token \
      --token=<enrollment-token> \
      --domain=<your-infisical-domain> \
      --pkcs11-module=/opt/fortanix/pkcs11/fortanix_pkcs11.so
    sudo systemctl start <gateway-name>
    ```
  </Tab>
  <Tab title="Foreground">
    ```bash
    infisical gateway start <gateway-name> \
      --enroll-method=token \
      --token=<enrollment-token> \
      --domain=<your-infisical-domain> \
      --pkcs11-module=/opt/fortanix/pkcs11/fortanix_pkcs11.so
    ```
  </Tab>
</Tabs>

Check the Gateway logs for a confirmation that the PKCS#11 module loaded.

## Step 3: Create the HSM Connector in Infisical

In **Certificate Manager > Settings > HSM Connectors**, click **Add HSM Connector** and fill in:

| Field | Value |
|-------|-------|
| **Name** | Slug-friendly identifier, e.g. `fortanix-prod`. |
| **Gateway** | The Gateway you started above (or a Pool that contains it). |
| **Slot label** | The Fortanix slot label from `pkcs11-tool --list-slots`. |
| **PIN** | The Fortanix application's PKCS#11 PIN. |
| **Key label prefix** | Optional. Useful if your DSM hosts keys for multiple environments. |

On submit Infisical runs a Verify round-trip through the Gateway. A wrong PIN, missing slot label, or unreachable Gateway is caught here and the Connector is not persisted.

## Troubleshooting

- **"slot not found" on Verify**: confirm the slot label matches what `pkcs11-tool --list-slots` reports. Fortanix slot labels are case-sensitive.
- **"login failed" on Verify**: re-check the PIN. The PIN is the Fortanix application's PKCS#11 PIN, not the DSM account password.
- **"gateway unreachable"**: the Gateway must be running with HSM support enabled. Check the Gateway log for the PKCS#11 module load line.
- **Slow operations**: if the Gateway host can't reach DSM over HTTPS with low latency, every sign call pays the round-trip. Run the Gateway in the same network as DSM whenever possible.
