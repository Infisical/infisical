Templates control what happens when someone connects to an account. How long can they stay connected? Do they need MFA? Should the session be recorded? These rules are defined in a template, and every account uses one.

Infisical provides **default templates** (`postgres`, `ssh`) with sensible settings, so you can start adding accounts right away. Create custom templates when you need different rules — for example, stricter settings for production databases.

## Default Templates

When you first use PAM, these templates are already available:

| Template | Type | Default Settings |
|----------|------|------------------|
| `postgres` | PostgreSQL | 1-hour sessions, recording enabled |
| `ssh` | SSH | 1-hour sessions, recording enabled |

These defaults work for many cases. If they fit your needs, you can skip straight to [adding accounts](/documentation/platform/pam/accounts/overview).

## Creating a Custom Template

If you need different rules — say, 4-hour sessions with MFA for production — create a custom template.

<Steps>
  <Step title="Navigate to Account Templates">
    Go to **Privileged Access Management → Account Templates** and click **Create Template**.
  </Step>
  <Step title="Set the basics">
    | Field | Description |
    |-------|-------------|
    | **Name** | A descriptive name (e.g., `prod-postgres`) |
    | **Type** | PostgreSQL or SSH |
    | **Description** | Optional context about when to use this template |
  </Step>
  <Step title="Configure access policy">
    These rules control the user experience when connecting:
    
    | Setting | What it does |
    |---------|--------------|
    | **Max Session Duration** | How long can someone stay connected |
    | **Require Reason** | Must they explain why they need access |
    | **Require MFA** | Must they verify their identity first |
  </Step>
  <Step title="Configure system settings">
    These settings control how the system behaves:
    
    | Setting | What it does |
    |---------|--------------|
    | **Recording Enabled** | Capture session activity |
    | **Recording Storage** | Where to store recordings (internal or your S3 bucket) |
    | **Password Constraints** | Minimum complexity for account passwords |
  </Step>
  <Step title="Save">
    Click **Create**. The template is now available when adding accounts.
  </Step>
</Steps>

## How Templates Work with Accounts

Every account references a template. When you create an account, you select which template it should use. The account then inherits all the rules from that template.

This means you can:
- Apply the same rules to many accounts by having them use the same template
- Change rules for all those accounts at once by editing the template
- Have different accounts in the same folder use different templates (if they need different rules)

<Warning>
Changes to templates apply immediately to all accounts using them. Be careful when tightening rules — it may affect active sessions.
</Warning>
