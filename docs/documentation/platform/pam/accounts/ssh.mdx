SSH accounts let you manage access to your Linux/Unix servers. Users connect through Web Access (a browser-based terminal) or CLI (a local proxy), and every session can be recorded.

SSH accounts support three authentication methods: password, public key, and certificate-based.

## Creating an Account

<Steps>
  <Step title="Open the folder">
    Go to **Folders** and click on the folder where you want to add the account.
  </Step>
  <Step title="Add account">
    Click **Add Account** and select **SSH**.
  </Step>
  <Step title="Select a template">
    Choose which template to use. The template determines session rules (duration, MFA, recording). You can use the default `ssh` template or a custom one.
  </Step>
  <Step title="Enter connection details">
    | Field | Description |
    |-------|-------------|
    | **Name** | A descriptive name (e.g., `prod-web-01`) |
    | **Host** | Server hostname or IP |
    | **Port** | SSH port (default: 22) |
    | **Username** | Linux username to connect as |
  </Step>
  <Step title="Choose authentication method">
    See the section below for each method.
  </Step>
  <Step title="Save">
    Click **Create**. For password and key auth, Infisical validates the connection. For certificate auth, you'll need to configure the server first.
  </Step>
</Steps>

## Authentication Methods

### Password

The simplest option — just provide the username and password.

| Field | Value |
|-------|-------|
| **Method** | Password |
| **Password** | The user's password |

<Note>
The server must have `PasswordAuthentication yes` in sshd_config.
</Note>

### Public Key

Authenticate with an SSH private key.

| Field | Value |
|-------|-------|
| **Method** | Public Key |
| **Private Key** | The private key (OpenSSH format) |

The corresponding public key must be in the user's `~/.ssh/authorized_keys` on the server.

### Certificate (Recommended)

The most secure option. Instead of storing static credentials, Infisical mints a short-lived certificate for each session.

| Field | Value |
|-------|-------|
| **Method** | Certificate |

After creating the account, you need to configure the server to trust Infisical's CA. The account detail page provides a setup script you can run on the server.

See the [SSH Certificate Auth guide](/documentation/platform/pam/guides/ssh-certificate-auth) for a full walkthrough.

## Connecting via Web Access

Web Access provides an interactive terminal in your browser.

To connect:
1. Open the account and click **Access**
2. Click **Connect in Browser**
3. A terminal opens in a new tab

You get a full shell — run commands, use vim, transfer files. The session is recorded.

To disconnect: type `exit` or close the tab.

## Connecting via CLI

The CLI starts a local proxy that you connect to with your SSH client.

```bash
infisical pam access /<folder>/<account>
```

For example:
```bash
infisical pam access /prod-servers/web-01
```

Output:
```
Proxy running on localhost:2222
Connect with: ssh deploy@localhost -p 2222

Session expires in 8 hours. Press Ctrl+C to disconnect.
```

Then connect:
```bash
ssh deploy@localhost -p 2222
```

<Tip>
No password or key needed — the proxy handles authentication.
</Tip>

### CLI Flags

| Flag | What it does |
|------|--------------|
| `--port <port>` | Use a specific local port |
| `--reason <reason>` | Provide an access reason (if required by template) |
