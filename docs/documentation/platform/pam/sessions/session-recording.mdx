When recording is enabled, PAM captures everything that happens during a session — every query, command, and response. This gives you a complete record for compliance audits and incident investigation.

## What Gets Recorded

The content depends on the account type:

**PostgreSQL sessions:**
- All SQL queries executed
- Query results and responses
- Timestamps for each action

**SSH sessions:**
- All commands entered
- Command output
- Timestamps for each action

Both Web Access and CLI sessions are recorded the same way.

## How Recording Works

Recording happens at the Gateway level:

1. Traffic flows through the Gateway
2. The Gateway captures and encrypts session events
3. Events upload to Infisical every 10 seconds
4. When the session ends, the full recording is available

This architecture means:
- **No data loss** — if connectivity drops, the Gateway caches locally and syncs when restored
- **Near real-time** — admins can see what's happening in active sessions
- **Always encrypted** — recordings are encrypted before upload

## Configuring Recording

Recording is configured at the **template level**. All accounts using a template inherit its recording settings.

| Setting | What it does |
|---------|--------------|
| **Recording Enabled** | Toggle recording on/off |
| **Storage Backend** | Where recordings are stored |

To change recording settings:
1. Go to **Account Templates** and open the template
2. Find **System Settings → Recording**
3. Toggle **Recording Enabled** and select the storage backend
4. Save

### Storage Options

| Backend | Description |
|---------|-------------|
| **Internal** | Stored in Infisical's managed database |
| **External S3** | Stored in your own S3 bucket |

For most cases, internal storage works fine. Use [External Storage](/documentation/platform/pam/sessions/external-storage) if you need control over data residency or retention.

## Viewing Recordings

1. Go to **Sessions** and click on a session
2. The session detail page shows the full log of activity
3. Use the search bar to find specific content

For active sessions, logs update every 10 seconds. For completed sessions, the full recording is available immediately.

## Best Practices

**Enable recording for production.** Session recordings are essential for compliance and incident investigation. Only disable for low-risk development environments where you don't need an audit trail.

**Use external S3 for regulated environments.** If you have data residency or retention requirements, store recordings in your own bucket where you control the data lifecycle.

**Configure log masking for sensitive data.** Templates can define patterns to redact from logs (like credit card numbers). This happens before storage, so sensitive data never gets recorded.
