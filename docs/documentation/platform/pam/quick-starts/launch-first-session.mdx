This quickstart walks you through connecting to a PostgreSQL database via PAM. You'll create a folder, add a database account, and launch a session.

<Note>
**Gateway required** — PAM connects to your databases and servers through a Gateway. If you haven't deployed one yet, [deploy a Gateway](/documentation/platform/gateways/gateway-deployment) first.
</Note>

## Prerequisites

- Admin access to PAM
- A deployed [Gateway](/documentation/platform/gateways/gateway-deployment) that can reach your database
- Database credentials (username and password)

## Steps

<Steps>
  <Step title="Create a Folder">
    Folders organize your accounts. Create one to hold your database.

    1. Go to **Privileged Access Management → Accounts**
    2. Click **Create Folder**
    3. Name it `dev-databases` (or whatever you like)
    4. Click **Create**
  </Step>
  <Step title="Add a Database Account">
    1. Open the folder you just created
    2. Click **Add Account**
    3. Select the `postgres` template
    4. Fill in your database connection details (host, port, credentials)
    5. Click **Create**
  </Step>
  <Step title="Launch a Session">
    1. Open the account you created
    2. Click **Access**
    3. Click **Connect in Browser**

    You'll see the Data Explorer (browse tables) and SQL Terminal (run queries).
  </Step>
</Steps>
