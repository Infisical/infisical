This guide shows you how to set up access request policies and request temporary or permanent access to sensitive environments and secret paths in Infisical.

## Prerequisites

- An Infisical project with at least one [environment](/documentation/platform/project) configured
- The **Admin** or **Member** role in the project (both roles can create access request policies)

## Set up an access request policy

<Steps>
  <Step title="Navigate to the Approvals page">
    In your project, go to the **Approvals** settings page where access request policies are managed.
  </Step>
  <Step title="Create a new access request policy">
    Click **Add Policy** and configure the following options:

    - **Name** — A descriptive name for the policy.
    - **Environment** — The environment this policy applies to.
    - **Secret Path** — The secret path (folder) this policy governs.
    - **Approvers** — The users or groups eligible to approve requests. You can assign approvers to sequential steps for multi-level approval chains.
    - **Approvals Required** — The number of approvals needed at each step.
    - **Enforcement Level** — Set to **Hard** (approvals are strictly required) or **Soft** (designated bypassers can use break-glass access without full approval).
    - **Bypassers** — Users or groups allowed to bypass the policy when enforcement is set to Soft.
    - **Allow Self-Approvals** — Whether requesters who are also approvers can approve their own requests. Enabled by default.
    - **Max Time Period** — The maximum duration of temporary access that can be requested under this policy (for example, `1h`, `30m`, `2d`). Leave empty to allow permanent access.

    ![Create Access Request Policy Modal](/images/platform/access-controls/create-access-request-policy.png)

    <Note>
      A step policy enables a sequential approval workflow in which approvals
      must follow the designated chain.
    </Note>

    ![Access Request Policies](/images/platform/access-controls/access-request-policies.png)
  </Step>
</Steps>

## Request access to a sensitive resource

<Steps>
  <Step title="Submit an access request">
    When you need access to an environment or secret path governed by a policy, submit an access request from the dashboard. Specify the permissions you need and the access duration.

    ![Access Request Create](/images/platform/access-controls/request-access.png)
  </Step>
  <Step title="Wait for approval">
    Your request appears in the dashboard for eligible approvers to review. Approvers also receive an email notification about the pending request.

    ![Access Request Dashboard](/images/platform/access-controls/access-requests-pending.png)
  </Step>
  <Step title="Approval or rejection">
    An eligible approver can approve or reject the access request.

    ![Access Request Bypass](/images/platform/access-controls/access-request-bypass.png)

    <Note>
      Approvers can edit the duration of an access request to reduce how long access is granted by clicking the **Edit** icon next to the duration.
    </Note>

    ![Edit Access Request](/images/platform/access-controls/edit-access-request.png)

    <Info>
      If the policy enforcement level is set to **Soft** and you are a designated bypasser, you can bypass the approval requirement and gain access immediately.
    </Info>
  </Step>
  <Step title="Access granted">
    Once the request is approved, you can access the requested resources. If the access is temporary, it expires automatically after the specified duration.
  </Step>
</Steps>
