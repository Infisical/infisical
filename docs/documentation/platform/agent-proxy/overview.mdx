The **Infisical Agent Proxy** is a credential broker for AI agents and untrusted code execution environments. It sits between your agents and the APIs they call, GitHub, Slack, LLM providers, or your own internal services, and attaches the real credential to each request on its way out, so the agent never holds one.

## How it works

```text
            Your private network                                Public internet

+----------+                 +-------------+                  +------------------+
| AI agent | --------------> | Agent Proxy | ---------------> |  api.slack.com   |
+----------+  request with a +-------------+ request with the |  api.github.com  |
             placeholder key        |        real credential  |  ...             |
                                    |                         +------------------+
                                    |  fetches the real
                                    |  credential values
                                    v
                             +-------------+
                             |  Infisical  |
                             +-------------+
```

Every request the agent makes follows the same three steps:

1. The agent calls an API with a dummy credential, or none. `HTTPS_PROXY` in its environment routes the call to the proxy.
2. The proxy matches the host to a [proxied service](/documentation/platform/agent-proxy/proxied-services), pulls the real secret from Infisical, and attaches it to the request.
3. The API sees a normal authenticated call. The agent never sees the credential.

## Why brokering

Traditional secrets management hands the credential back to your application. An application uses it exactly as programmed; an agent does whatever its context convinces it to do. Once a key enters an agent's environment:

- Prompt injection is an unsolved attack class. Any credential the agent can read, an attacker can exfiltrate.
- Anything in the agent's environment is candidate context, and context leaves your infrastructure on every model call.
- Propagation is untracked and irreversible. There is no way to know where copies of the key exist, and the only remediation is rotation, which breaks every legitimate consumer of the key.

Brokering removes the problem at the source: the key never enters the agent, so there is nothing to leak, spread, or chase down.

## What you work with

| Object | What it is |
| --- | --- |
| **Proxied service** | A config that maps a host to the secret the proxy applies, static or [dynamic](/documentation/platform/agent-proxy/proxied-services#using-dynamic-secrets). Defined next to your secrets in the dashboard. |
| **Agent proxy** | The forward proxy in the [Infisical CLI](/cli/overview) that runs in your network and attaches credentials to outbound requests. |
| **Agent** | Any program that respects `HTTPS_PROXY`: Claude Code, Codex, OpenClaw, Hermes, or any other untrusted workload. |

<Warning>
  The Agent Proxy is distinct from similarly named Infisical components:

  - The [Infisical Agent](/integrations/platforms/infisical-agent) fetches secrets to disk for applications.
  - The [Infisical Proxy](/integrations/platforms/infisical-proxy) caches the Infisical API.
  - [Gateways](/documentation/platform/gateways/overview) let Infisical reach into private networks.
</Warning>
