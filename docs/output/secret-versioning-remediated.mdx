Every time a secret is created or updated, Infisical records a new version of that secret. Each version is assigned an incrementing version number, starting at 1.

## Viewing Secret Versions

Secret versions can be accessed visually by opening the [secret sidebar](/documentation/platform/secrets-mgmt/project#drawer).

![secret versioning overview](../../images/platform/secret-versioning-overview.png)
![secret versioning](../../images/platform/secret-versioning.png)

## Retrieving Versions via API

You can retrieve a specific version of a secret by passing the `version` query parameter to the [Get a Secret](/api-reference/endpoints/secrets/read) endpoint. When omitted, the latest version is returned.

## Version Lifecycle

- **Create**: When a secret is first created, it is assigned version 1.
- **Update**: Each update increments the version number by 1 and creates a new version record.
- **Delete**: When a secret is deleted, a version record of the deletion is preserved.

All version records are retained for use with [Point-in-Time Recovery](/documentation/platform/pit-recovery).

## Rolling Back

Currently, you can manually roll back to a previous secret version by copying its value from the version history and pasting it into the secret's current value field. This creates a new version at the top of the stack.

For automated rollback of an entire folder or environment to a previous state, use [Point-in-Time Recovery](/documentation/platform/pit-recovery).

<Note>
  Individual secret-level automatic rollback is not yet available. Use
  Point-in-Time Recovery to restore secrets at the folder or environment level.
</Note>
